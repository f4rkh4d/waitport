# contributing

thanks for wanting to help.

## bug reports

open an issue. include what you ran, what you expected, and what happened.
screenshots or terminal output are great.

## pull requests

1. fork + branch.
2. if the change adds behavior, add a test.
3. run the test suite locally first.
4. keep the pr focused. one idea per pr.
5. describe the _why_, not just the _what_.

no signed commits required. no forms to fill. just keep it small and clear.
